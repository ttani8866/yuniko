# 記事生成ワークフロー

ユーザーの曖昧な入力から、Zettelkastenルールに準拠した長文Obsidian記事を生成します。

## 1. 入力分析とトピック拡張
- ユーザーの「もやっとした」リクエストを分析
- 核となるコンセプトと不足しているコンテキストを特定
- 1万文字を達成するための関連テーマをブレインストーミング

## 2. 深掘りリサーチ（Web検索）
// turbo
- `search_web` で歴史的背景、技術的詳細、将来の展望を調査
- `search_web` でトピックに関連する興味深いエピソードや感情的なストーリーを収集
- 1万文字の物語を支えるだけの材料を確保

## 3. 構成とドラフト作成
- 収集した情報を一貫した構造に整理
- ファイル名を決定: `YYYY-MM-DD_タイトル.md`
- Zettelkastenリンクとタグを計画

## 4. 記事執筆
- `obsidian_writer` ペルソナ（感情的、主観的、プロ）を使用
- 出力先: `テツの脳みそ/01/YYYY-MM-DD_タイトル.md`
- **重要**: 必ず **10,000文字以上** で執筆
- Frontmatter（created, tags, aliases）を含める
- Obsidian形式の内部リンク `[[リンク]]` を活用
- 文末に `#ハッシュタグ` と `思考の接続点` セクションを追加

## 5. 最終レビュー
- ファイルの存在を確認
- 文字数を確認（1万文字以上か）
- Zettelkastenリンクの整合性を確認
